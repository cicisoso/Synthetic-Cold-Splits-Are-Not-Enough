\section{Benchmark Setup}

Our goal is to test whether a compact DTI model panel preserves its ranking across synthetic cold-start regimes and real distribution shifts. The replacement panel used in the revised manuscript contains three models: \texttt{base}, a fixed-feature MLP reference system built from RDKit Morgan fingerprints and hashed protein k-mer features; \texttt{DTI-LM}, a pooled ChemBERTa+ESM2 baseline aligned to recent literature; and \texttt{HyperPCM}, a task-conditioned pooled-LM baseline. We intentionally keep this external panel small so that benchmark effects are not obscured by a large architecture zoo while still avoiding dependence on internal probe models. Because the anchor system is central to every ranking claim, Appendix Table~\ref{tab:base-spec} gives its exact feature construction, architecture, parameter count, and training recipe.

We evaluate this panel on two benchmark groups. The first is a synthetic reference panel based on \texttt{BindingDB\_Kd}, with three split definitions: \texttt{unseen\_drug}, \texttt{blind\_start}, and \texttt{unseen\_target}. The second is a real-OOD screening panel. Our primary real-OOD benchmark is the BindingDB patent split (\texttt{patent\_temporal}), which uses a year-based temporal partition and is evaluated over three seeds. We also include \texttt{BindingDB\_Ki / unseen\_target} and \texttt{DAVIS / unseen\_target} as supporting shifted target-cold benchmarks that test whether the same external model family remains stable across assay and dataset shifts. Table~\ref{tab:taxonomy} summarizes the shift taxonomy and split statistics.

We report \AUPRC{} as the primary metric and \AUROC{} as a secondary metric. \AUPRC{} is emphasized because class balance and ranking behavior are central to the benchmark claims in this paper. For the replacement panel, we report matched three-seed results on the synthetic reference panel and the promoted patent benchmark. We first run a transparent seed0 screening pass on \texttt{patent\_temporal}, \texttt{blind\_start}, and \texttt{unseen\_target}, then promote the strongest or most informative recent baselines to matched three-seed evaluation. \texttt{DTI-LM} is promoted because it is the strongest seed0 and 3-seed recent baseline on the primary temporal/provenance benchmark \texttt{patent\_temporal}, while \texttt{HyperPCM} is kept because it is competitive on the synthetic reference panel and therefore provides a meaningful replacement for the internal probe lines. Appendix Table~\ref{tab:recent-screening} makes this promotion logic explicit. We further add paired bootstrap uncertainty support for the headline external-panel comparisons against \texttt{base}, including the promoted patent comparison and the synthetic reference-panel contrasts where matched three-seed results are available. Appendix Table~\ref{tab:bootstrap-summary} collects these intervals.

To make the benchmark reusable, each split is exported as a canonical resource bundle with manifests, split tables, cached identifiers, and a standalone CSV-based evaluator. This resource layer allows external predictions to be scored against the exact same test partitions used in the paper. Appendix Table~\ref{tab:resource-release} enumerates the released artifacts and how they are consumed by the evaluator.

The patent benchmark requires several setup choices that are documented explicitly. We use a local year-based split of \texttt{BindingDB\_patent}, with training years before the validation year, a held-out validation year, and a later temporal test set. Because this benchmark also changes provenance and assay composition, we describe it as temporal/provenance OOD rather than pure time shift. We also use a patent-specific affinity conversion rather than reusing the standard \texttt{pKd} transform, because the patent benchmark stores affinities in a different log-scale convention. Appendix Table~\ref{tab:patent-spec} summarizes the exact year boundaries, split counts, label construction, overlap policy, and seed semantics for both \texttt{patent\_temporal} and the alternative-cutoff \texttt{patent\_temporal\_v2017}. BindingDB-derived panels are anchored in the BindingDB resource \citep{liu2007bindingdb}, while DAVIS provides an external kinase-focused support benchmark \citep{davis2011kinase}.

Finally, for diagnostic analysis, we use the saved overlap score from each test prediction to compute overlap buckets that summarize how strongly each test example resembles the training support. We report benchmark taxonomy and split statistics, including test-set size and prevalence, so that \AUPRC{} is interpreted within benchmark rather than across benchmark families.
