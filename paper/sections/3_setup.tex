\section{Benchmark Setup}

Our goal is to test whether a compact DTI model panel preserves its ranking across synthetic cold-start regimes and real distribution shifts. The core internal panel contains three models: \texttt{base}, a plain encoder that serves as the reference system; \texttt{RAICD}, a retrieval-augmented model that aggregates support from similar seen entities; and \texttt{FTM sparse}, a functional target-memory model designed to improve target-cold generalization. We intentionally keep this internal panel small so that benchmark effects are not obscured by a large architecture zoo. We then expand the panel with recent pooled-LM baselines in a second stage. \texttt{DTI-LM} is promoted as the primary recent baseline in the main table, while \texttt{HyperPCM} is run on the same axes as a follow-up check and summarized outside the main topline table because it does not change the main ranking.

We evaluate this panel on two benchmark groups. The first is a synthetic reference panel based on \texttt{BindingDB\_Kd}, with three split definitions: \texttt{unseen\_drug}, \texttt{blind\_start}, and \texttt{unseen\_target}. The second is a real-OOD screening panel. Our primary real-OOD benchmark is the BindingDB patent split (\texttt{patent\_temporal}), which uses a year-based temporal partition and is evaluated over three seeds. We also include \texttt{BindingDB\_Ki / unseen\_target} and \texttt{DAVIS / unseen\_target} as supporting shifted target-cold benchmarks that test whether the synthetic \texttt{unseen\_target} gain of \texttt{FTM sparse} transfers across assay and dataset shifts. We report \AUPRC{} as the primary metric and \AUROC{} as a secondary metric. \AUPRC{} is emphasized because class balance and ranking behavior are central to the benchmark claims in this paper. For the internal panel, we report matched three-seed results on all benchmark axes shown in the main text and appendix. For recent baselines, we first run a transparent seed0 screening pass on \texttt{patent\_temporal}, \texttt{blind\_start}, and \texttt{unseen\_target}, then promote the strongest or most informative candidates to matched three-seed evaluation. \texttt{DTI-LM} is promoted in the main table because it is the strongest seed0 and 3-seed recent baseline on the primary real-OOD benchmark \texttt{patent\_temporal}, while \texttt{HyperPCM} is kept as an additional recent-baseline check because it is competitive on some seed0 synthetic axes without changing the main patent conclusion. Appendix Table~\ref{tab:recent-screening} makes this promotion logic explicit. We further add paired bootstrap uncertainty support for every comparison used in a headline ranking claim: \texttt{RAICD} versus \texttt{base} on synthetic \texttt{blind\_start}, \texttt{FTM} versus \texttt{base} on synthetic \texttt{unseen\_target}, \texttt{base} versus \texttt{RAICD} or \texttt{FTM} on patent temporal, and the supporting \texttt{base} comparisons on \texttt{BindingDB\_Ki} and \texttt{DAVIS}. Appendix Table~\ref{tab:bootstrap-summary} collects these intervals.

To make the benchmark reusable, each split is exported as a canonical resource bundle with manifests, split tables, cached identifiers, and a standalone CSV-based evaluator. This resource layer allows external predictions to be scored against the exact same test partitions used in the paper. Appendix Table~\ref{tab:resource-release} enumerates the released artifacts and how they are consumed by the evaluator.

The patent benchmark requires several setup choices that are documented explicitly. We use a local year-based split of \texttt{BindingDB\_patent}, with training years before the validation year, a held-out validation year, and a later temporal test set. Because this benchmark also changes provenance and assay composition, we describe it as temporal/provenance OOD rather than pure time shift. We also use a patent-specific affinity conversion rather than reusing the standard \texttt{pKd} transform, because the patent benchmark stores affinities in a different log-scale convention. Appendix Table~\ref{tab:patent-spec} summarizes the exact year boundaries, split counts, label construction, overlap policy, and seed semantics for both \texttt{patent\_temporal} and the alternative-cutoff \texttt{patent\_temporal\_v2017}. BindingDB-derived panels are anchored in the BindingDB resource \citep{liu2007bindingdb}, while DAVIS provides an external kinase-focused support benchmark \citep{davis2011kinase}.

Finally, for diagnostic analysis, we use the saved overlap score from each test prediction to compute overlap buckets that summarize how strongly each test example resembles the training support. We report benchmark taxonomy and split statistics, including test-set size and prevalence, so that \AUPRC{} is interpreted within benchmark rather than across benchmark families.
